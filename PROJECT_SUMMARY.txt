WHAT IS THEME SWITCHER?
================================================================================

A Visual Studio Code extension that automatically rotates through installed
color themes at configurable hourly intervals, helping developers escape
monotony and enjoy a more colorful coding environment.

Key Features:
- Automatic hourly theme switching
- Manual switch command (Ctrl+Shift+P)
- Pause/Resume controls
- Configurable polling intervals
- Full Settings UI support
- Zero external dependencies
- Secure and lightweight

CONFIGURATION REFERENCE
================================================================================

Settings (in settings.json):

1. themeSwitcher.switchIntervalHours (number, default: 1)
   - Hours between automatic theme switches
   - Minimum: 1 hour
   - Examples: 1, 8, 12, 24

2. themeSwitcher.pollIntervalMinutes (number, default: 20)
   - Minutes between polling checks
   - Minimum: 1 minute
   - Typical: 10-20 minutes (balance overhead vs. responsiveness)

3. themeSwitcher.enabled (boolean, default: true)
   - Enable/disable extension globally
   - Can use pause/resume commands instead

Example (colorful workday):
{
  "themeSwitcher.switchIntervalHours": 1,
  "themeSwitcher.pollIntervalMinutes": 15,
  "themeSwitcher.enabled": true
}

================================================================================
HOW IT WORKS
================================================================================

Flow:
1. Extension activates on VS Code startup
2. Reads configuration (hours, minutes)
3. Checks if switch is due immediately
4. Sets up polling timer
5. Every pollIntervalMinutes:
   - Check how many hours since last switch
   - If >= switchIntervalHours, switch to next theme
   - Show notification
6. Loop continues until extension stops

Hour-Based Calculation:
- Stores lastSwitchTimestamp when theme changes
- Calculates hours elapsed = (now - lastSwitchTime) / (1000 * 60 * 60)
- Compares with switchIntervalHours threshold
- Minimal CPU/battery impact (only checks every N minutes)

2. theme-switcher.pause
   - Pauses automatic switching
   - User can still manually switch
   - State persists until resume

3. theme-switcher.resume
   - Resumes automatic switching
