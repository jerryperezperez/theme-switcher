## üöÄ Common Workflows

### "I want to use Theme Switcher"
1. Start with [README.md](./README.md)
2. Configure in Settings UI
3. Check [README.md - Troubleshooting](./README.md#-troubleshooting) if issues

### "I want to develop the extension"
1. Follow [QUICKSTART.md](./QUICKSTART.md) (5 minutes)
2. Read [DEVELOPMENT.md](./DEVELOPMENT.md) for deep dive
3. Check [CONTRIBUTING.md](./CONTRIBUTING.md) before submitting code

### "I found a bug"
1. Check [README.md - Troubleshooting](./README.md#-troubleshooting)
2. Report on GitHub Issues with details
3. Follow [CONTRIBUTING.md - Reporting Bugs](./CONTRIBUTING.md#reporting-bugs)

### "I want to suggest a feature"
1. Follow [CONTRIBUTING.md - Suggesting Features](./CONTRIBUTING.md#suggesting-features)
2. Create a GitHub Issue with enhancement label

### "I found a security issue"
1. Read [SECURITY.md](./SECURITY.md)
2. DO NOT open a public issue
3. Report privately via email (see SECURITY.md)

### "I want to publish to Marketplace"
1. Check [PROJECT_CHECKLIST.md - Pre-Publication](./PROJECT_CHECKLIST.md#-pre-publication-checklist)
2. Follow [PUBLISHING.md](./PUBLISHING.md)
3. Or see quick version in [QUICKSTART.md - Publishing](./QUICKSTART.md#-publishing)

### "I want to contribute code"
1. Fork the repository
2. Follow [CONTRIBUTING.md - Development Setup](./CONTRIBUTING.md#development-setup)
3. Follow [QUICKSTART.md](./QUICKSTART.md) for development
4. Review [CONTRIBUTING.md - Code Style](./CONTRIBUTING.md#code-style)
5. Create pull request per [CONTRIBUTING.md - Pull Requests](./CONTRIBUTING.md#pull-requests)
